# ASON 格式规范 v1.0

> **ASON** = Array-Schema Object Notation

## 1. 设计理念

- **Schema 与数据分离**：先声明结构，后填充数据
- **最小化冗余**：字段名只声明一次，数据部分不重复 key
- **单行友好**：所有格式天然支持压缩为一行
- **Token 高效**：专为减少 LLM token 消耗设计

## 2. 基本语法

```
schema:data
```

- `schema`：定义数据结构（字段名、嵌套关系）
- `data`：纯数据值，按 schema 顺序排列

## 3. 基本类型

| 类型 | 示例 | 说明 |
|------|------|------|
| 整数 | `42`, `-100` | 无小数点的数字 |
| 浮点 | `3.14`, `-0.5` | 带小数点的数字 |
| 字符串 | `hello`, `"world"` | 可选引号 |
| 布尔 | `true`, `false` | 小写 |
| 空值 | _(空白)_ | 字段位置留空 |
| 空字符串 | `""` | 显式空字符串 |

### 3.1 字符串规则

ASON 支持两种字符串形式：

| 形式 | 示例 | 行为 |
|------|------|------|
| 无引号 | `hello world` | 自动 trim 首尾空格 |
| 有引号 | `" hello "` | 保留原样（包括空格） |

**使用引号的场景：**
- 保留首尾空格：`" hello "`
- 前导零：`"001234"`（邮编）
- 强制字符串：`"true"`, `"123"`
- 空字符串：`""`

## 4. 转义规则

当字符串值包含特殊字符时，需要转义：

| 字符 | 转义形式 | 说明 |
|------|----------|------|
| `,` | `\,` | 逗号 |
| `(` | `\(` | 左括号 |
| `)` | `\)` | 右括号 |
| `[` | `\[` | 左方括号 |
| `]` | `\]` | 右方括号 |
| `"` | `\"` | 双引号 |
| `\` | `\\` | 反斜杠 |
| 换行 | `\n` | 换行符 |
| 制表 | `\t` | 制表符 |

**注意**：
- 无引号字符串中，必须转义 `,()[]`
- 有引号字符串中，只需转义 `"` 和 `\`

## 5. 注释

支持块注释，使用 `/* */` 语法：

```
/* 这是一个用户列表 */
{name,age}:(Alice,30),(Bob,25)
```

多行注释：
```
/*
  用户数据
  最后更新: 2024-01-01
*/
{name,age}:
  (Alice,30),
  (Bob,25)
```

注释可以出现在任意位置（schema 前、中、后，数据中）：
```
{name /* 用户名 */, age}:(Alice, /* 年龄 */ 30)
```

## 6. 语法规则

### 6.1 单个对象

```
{name,age}:(Alice,30)
```
→ `{name: "Alice", age: 30}`

### 6.2 对象数组（同结构多条数据）

```
{name,age}:(Alice,30),(Bob,25),(Charlie,35)
```
→ 3 个对象的数组

### 6.3 空值/可选字段

```
{name,age,email}:(Alice,30,)
```
→ `{name: "Alice", age: 30, email: null}`

### 6.4 嵌套对象

```
{name,addr{city,zip}}:(Alice,(NYC,10001))
```
→ `{name: "Alice", addr: {city: "NYC", zip: 10001}}`

### 6.5 对象包含简单数组

```
{name,scores[]}:(Alice,[90,85,92])
```
→ `{name: "Alice", scores: [90, 85, 92]}`

### 6.6 对象包含对象数组

```
{team,users[{id,name}]}:(Dev,[(1,Alice),(2,Bob)])
```
→ `{team: "Dev", users: [{id: 1, name: "Alice"}, {id: 2, name: "Bob"}]}`

### 6.7 纯数组（无 schema）

```
[1,2,3]
```
→ `[1, 2, 3]`

### 6.8 数组中的数组

```
[[1,2],[3,4]]
```
→ `[[1, 2], [3, 4]]`

### 6.9 空数组

```
[]
```

### 6.10 空对象

```
()
```

### 6.11 混合类型数组

```
[1,hello,true,3.14]
```
→ `[1, "hello", true, 3.14]`

### 6.12 复杂嵌套示例

```
{company,employees[{id,name,skills[]}],active}:(ACME,[(1,Alice,[rust,go]),(2,Bob,[python])],true)
```

**解析结果：**
- `company` = `"ACME"`
- `employees` = 包含 2 个对象的数组
  - `{id: 1, name: "Alice", skills: ["rust", "go"]}`
  - `{id: 2, name: "Bob", skills: ["python"]}`
- `active` = `true`

### 6.13 字符串处理示例

```
{name,city,zip,note}:
  (Alice, New York, "001234", hello world),
  (Bob, "  Los Angeles  ", 90210, "say \"hi\"")
```

**解析结果：**
| 字段 | Alice 行 | Bob 行 |
|------|----------|--------|
| name | `"Alice"` (trim) | `"Bob"` (trim) |
| city | `"New York"` (trim) | `"  Los Angeles  "` (引号保留) |
| zip | `"001234"` (引号保留前导零) | `90210` (整数) |
| note | `"hello world"` (trim) | `"say \"hi\""` (引号+转义) |

## 7. 语法速查表

| 元素 | Schema 语法 | 数据语法 |
|------|-------------|----------|
| 对象 | `{field1,field2}` | `(val1,val2)` |
| 简单数组 | `[]` | `[v1,v2,v3]` |
| 对象数组 | `[{f1,f2}]` | `[(v1,v2),(v3,v4)]` |
| 嵌套对象 | `{f1,f2{f3,f4}}` | `(v1,(v3,v4))` |
| 空值 | - | _(空白)_ |
| 空数组 | - | `[]` |
| 空对象 | - | `()` |

## 8. 详细规则

### 8.1 类型解析优先级

解析值时按以下顺序尝试：

1. **空白** → `null`
2. **布尔** → `true` 或 `false`（小写）
3. **整数** → 匹配 `-?[0-9]+`
4. **浮点** → 匹配 `-?[0-9]+\.[0-9]+`
5. **字符串** → 其他所有情况

示例：
| 值 | 解析为 |
|----|--------|
| _(空白)_ | `null` |
| `true` | 布尔 `true` |
| `123` | 整数 `123` |
| `3.14` | 浮点 `3.14` |
| `hello` | 字符串 `"hello"` |
| `123abc` | 字符串 `"123abc"` |

### 8.2 空值与空字符串

| ASON 写法 | 解析结果 |
|-----------|----------|
| `{name,age}:(Alice,)` | `age = null` |
| `{name,age}:(Alice,"")` | `age = ""` (空字符串) |

示例：
```
{name,bio}:(Alice,)         /* bio = null */
{name,bio}:(Alice,"")       /* bio = "" (空字符串) */
```

### 8.3 顶层结构识别

顶层只有 **三种形式**，根据首字符判断：

| 首字符 | 类型 | 示例 |
|--------|------|------|
| `{` | 带 Schema 的对象/对象数组 | `{name,age}:(Alice,30)` |
| `[` | 纯数组 | `[1,2,3]` |
| 其他 | 纯值（自动推断类型） | `42`, `true`, `hello` |

**注意**：顶层禁止使用 `()`，`()` 只能出现在 schema 之后的数据部分。

### 8.4 字段名规则

- **允许字符**：`a-z`, `A-Z`, `0-9`, `_`
- **可以以数字开头**：`1st`, `2name` 合法
- **禁止字符**：`,`, `{`, `}`, `[`, `]`, `(`, `)`, `:`

### 8.5 空格处理规则

| 位置 | 规则 |
|------|------|
| Schema 中 | 忽略所有空格 |
| 数据中 | 保留空格（作为字符串一部分） |
| 逗号后 | 忽略前导空格 |

示例：
```
{name, age}:(Alice Smith, 30)
```
等价于：
```
{name,age}:(Alice Smith,30)
```
→ `{"name": "Alice Smith", "age": 30}`

### 8.6 多行格式

支持换行和缩进，解析时视为空白处理：

```
{name,age}:
  (Alice,30),
  (Bob,25),
  (Charlie,35)
```

等价于：
```
{name,age}:(Alice,30),(Bob,25),(Charlie,35)
```

### 8.7 负数规则

负号 `-` 必须紧跟数字，中间不能有空格：

| 写法 | 解析为 |
|------|--------|
| `-123` | 整数 `-123` ✓ |
| `- 123` | 字符串 `"- 123"` ✗ |
| `-3.14` | 浮点 `-3.14` ✓ |

### 8.8 数据对齐规则（严格模式）

**数据项数量必须与 Schema 字段数严格一致。**

| Schema | 数据 | 结果 |
|--------|------|------|
| `{a,b,c}` | `(1,2,3)` | ✓ 正确 |
| `{a,b,c}` | `(1,2)` | ✗ 错误：缺少字段 |
| `{a,b,c}` | `(1,2,3,4)` | ✗ 错误：字段过多 |
| `{a,b,c}` | `(1,,3)` | ✓ 正确：b = null |

**原因**：ASON 是位置敏感格式，字段数不匹配会导致数据错位，必须在解析时报错。

## 9. 完整语法 BNF（简化版）

```bnf
ason        ::= object_expr | array | value

object_expr ::= schema ":" object_list
schema      ::= "{" field_list "}"
field_list  ::= field ("," field)*
field       ::= identifier | identifier schema | identifier "[]" | identifier "[" schema "]"
identifier  ::= [a-zA-Z0-9_]+

object_list ::= object ("," object)*
object      ::= "(" value_list ")"
value_list  ::= value? ("," value?)*

array       ::= "[" array_items? "]"
array_items ::= array_item ("," array_item)*
array_item  ::= value | object | array

value       ::= boolean | number | quoted_str | plain_str | null
boolean     ::= "true" | "false"
number      ::= integer | float
integer     ::= "-"?[0-9]+
float       ::= "-"?[0-9]+"."[0-9]+

quoted_str  ::= '"' (escaped_char | [^"\\])* '"'
plain_str   ::= (plain_char | escaped_char)+
plain_char  ::= [^,()[\]"\\]                   /* 不含分隔符和引号 */
escaped_char::= "\\" | "\," | "\(" | "\)" | "\[" | "\]" | "\"" | "\n" | "\t"

null        ::= (empty)
comment     ::= "/*" (any_char)* "*/"
```

**注**：
- 注释 `/* ... */` 可以出现在任意位置，解析时忽略
- `plain_str` 解析后自动 trim 首尾空格
- `quoted_str` 保留原样（包括空格）

## 10. 实现注意事项

### 10.1 解析器实现要点

1. **非贪婪匹配**：`plain_str` 解析时，分隔符 `,()[]` 优先于字符串内容
2. **Lookahead**：遇到潜在分隔符时，先检查是否为转义字符
3. **注释剥离**：建议在词法分析阶段先移除所有 `/* */` 注释
4. **流式解析**：Schema 解析后构建 Access Map，数据部分按索引填充

### 10.2 错误处理

| 错误类型 | 示例 | 处理 |
|----------|------|------|
| 字段数不匹配 | `{a,b}:(1,2,3)` | 抛出错误，报告位置 |
| 未闭合引号 | `("hello)` | 抛出错误 |
| 未闭合括号 | `{a,b}:(1,2` | 抛出错误 |
| 未闭合注释 | `/* comment` | 抛出错误 |
| 非法转义 | `\x` | 抛出错误或保留原样 |
